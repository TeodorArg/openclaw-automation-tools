## Long Description

Turn vague "commit this and open a PR" requests into a bounded, branch-aware shipping flow for OpenClaw.

`openclaw-host-git-workflow` helps agents move real repo work on a host machine through doctor, branch prep, push, PR, checks, merge, and `main` sync without falling back to arbitrary git shell chaos.

It is built for people who want safer host-side Git delivery, fewer wrong-branch and wrong-repo mistakes, and clearer preflight feedback when SSH, GitHub auth, or host readiness is not actually ready.

## Key Bullets

- Adds a disciplined Git workflow instead of one-off shell commands from chat
- Resolves the right repo, branch, and host context before making changes
- Guides the agent through preflight, branch entry, commit prep, push, PR creation, and merge
- Reduces avoidable Git mistakes with explicit checkpoints and confirmation points
- Keeps host-side Git work predictable, repeatable, and easier to audit

## Release Post Markdown

```md
# Host Git Workflow: Branch to Merge, Safely

Ship branch -> PR -> checks -> merge safely from chat.

If you use OpenClaw on a real host repo, this plugin gives your agent a guarded shipping lane instead of ad-hoc git shell commands.

It helps with:
- doctor and preflight before push/PR
- branch-aware and repo-aware execution
- PR creation from the current non-main branch
- waiting for checks before merge
- safe merge and sync of `main`
- concrete remediation when GitHub SSH/auth is not ready

Why it matters:
- fewer wrong-branch and wrong-repo mistakes
- cleaner PR flow
- more predictable host-side git execution
- better auditability than re-explaining git steps in every chat

Primary entrypoint:
- `send_to_git`
- `отправь в гит`

Install:
```bash
openclaw plugins install clawhub:@openclaw/openclaw-host-git-workflow
```

Then enable it with your host repo path and host node selector, and ask OpenClaw to `send_to_git`.
```
